# Статус MCP-усилений для сайта МИИИИПС

## Что уже подготовлено
- сайт подготовлен для публикации под кастомным доменом `miiiips.ru` через GitHub Pages;
- есть рабочий GitHub-репозиторий;
- формы идут по маршруту email + Google Sheets;
- есть аудиторы `site_smoke_test.py` и `site_full_audit.py`;
- есть живая новостная лента, Timepad/Telegram/YouTube слой и support widget;
- сайт уже разделён на публичные страницы, ролевые кабинеты и служебные маршруты.

## Что подготовлено специально под Browser MCP
- стабильные публичные URL страниц;
- явные пользовательские сценарии: лекции, курс ЭИ, публикации, гранты, кабинеты, новости;
- формы с видимыми статусами отправки;
- PDF и документы с рабочими ссылками;
- mobile-чувствительные страницы выделены в отдельные сценарии.

## Что подготовлено специально под GitHub MCP
- отдельный репозиторий сайта;
- история правок и понятные коммиты;
- workflow обновления публичных фидов;
- GitHub Pages как постоянный канал публикации.

## Что подготовлено под Google / YouTube слой
- Google Sheets и Gmail уже участвуют в форме заявок;
- есть публичный JSON-слой с источниками лекций, контактами и видео;
- есть отдельный генератор живых фидов `refresh_public_feeds.py`.

## Что подготовлено под домен и наблюдаемость
- `CNAME` для `miiiips.ru`;
- `robots.txt` и `sitemap.xml`;
- `assets/data/site-settings.json`;
- `assets/js/site-observability.js`;
- базовые hooks для GA4, Search Console и будущего InsightfulPipe.

## Что ещё требует внешней настройки в среде Codex
1. Browser MCP
2. GitHub MCP
3. при необходимости Google/YouTube MCP или отдельные коннекторы

## Что останется сделать после включения MCP
- прогнать auto smoke по всем страницам;
- добавить автоматические mobile regression checks;
- делать screenshot baseline после каждой волны правок;
- закрепить сценарии для лекций, Timepad, формы и кабинетов как обязательный набор проверок.
