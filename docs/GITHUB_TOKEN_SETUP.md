# GitHub token setup for autonomous repository work

Когда захотим, чтобы Codex реже просил вход через браузер и мог стабильнее работать с GitHub, можно создать Personal Access Token.

## Зачем нужен token
- push/pull без ручного браузерного подтверждения
- работа с API GitHub
- автоматизация release/deploy задач
- удобная настройка репозиториев и GitHub Pages

## Минимальный безопасный набор прав
Для классического token:
- `repo`
- при необходимости позже: `workflow`

## Как использовать
1. Создать token в GitHub
2. Сохранить его локально как секрет, а не в репозитории
3. Настроить Git credential helper или переменную окружения

## Важно
Token не хранится в сайте и не коммитится в Git.
