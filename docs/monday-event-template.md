## Короткий анонс
- 2–4 предложения для посетителя сайта.
- Формулировка спокойная, ясная, без служебных слов и без внутренней кухни.

## Порядок блоков на странице
1. Верхняя шапка института.
2. Левая колонка со статусом, датой, спикером и форматом.
3. Главный заголовок и краткий анонс.
4. Основная фотография лекции.
5. Блок «О чём была лекция» или аналогичный смысловой блок.
6. Блок «Ключевые положения» с 3–4 крупными карточками.
7. Блок «Связанные маршруты».
8. Подвал с контактами.
9. Внутренний follow-up пакет для Telegram: не на публичной странице, а в event package.

## О чём была лекция
- 2–3 абзаца с нормальной человеческой подачей.
- Тема, контекст, смысл разговора и мягкий практический вывод.

## Ключевые темы
- `{{theme_1}}`
- `{{theme_2}}`
- `{{theme_3}}`
- `{{theme_4}}`

## Смысловой контур
- Короткий итог о том, чему посвящена лекция и почему она важна для института.
- Если данных мало, писать нейтрально и не выдумывать.

## Что должно использоваться
- Фото лекции как главный визуальный якорь.
- Текст, собранный из расшифровки и смысловой выжимки.
- Ключевые положения, написанные человеческим, но аккуратным институтским языком.
- Если есть mp3, его смысл должен попасть в основной текст и в тезисы, а не в служебные подписи.
- После публикации live-страницы должен быть подготовлен Telegram follow-up пост для канала Татьяны Мунн.

## Что нельзя менять
- Не переставлять левую и правую колонки местами.
- Не добавлять служебные подсказки для разработчика в публичный текст.
- Не добавлять на страницу блок «Материалы» с техтекстом вроде «архив», «подготовлено», «конспект» и т. п.
- Не превращать блоки в краткие плашки без смысловой нагрузки.
- Не менять шаблон навигации у понедельничных лекций без отдельного решения.

## Telegram Follow-Up После Публикации

Этот шаг обязателен для каждой опубликованной понедельничной лекции, но выполняется только после проверки live-страницы.

Порядок:
1. Проверить, что событие опубликовано на `miiiips.ru`, есть в `assets/data/events.json`, имеет рабочую HTML-страницу, фото, calendar `.ics`, page manifest и AEO package.
2. Проверить, что live-страница фактически открывается, фото отдаётся `200`, видео-анонс не является заглушкой, а публичный HTML не ведёт пользователя на Google Drive.
3. Из расшифровки аудио подготовить средний Telegram-пост от лица Татьяны Мунн: 900–1600 знаков, без технических слов, без внутренних путей, без Drive-ссылок.
4. В посте использовать фото лекции из event package / опубликованного site asset. Видео в Telegram не отправлять, если пользователь не попросил отдельно.
5. Проверить Moonn paid lecture route:
   - если конкретная лекция уже есть на `moonn.ru`, использовать прямую ссылку на неё;
   - если конкретной лекции нет, остановиться до публикации поста, создать/обновить страницу лекции в Tilda/Moonn через утверждённый доступ и только потом вставить прямую ссылку;
   - `https://moonn.ru/lectures1` не считать прямой оплатой по умолчанию: это общий архив/витрина и годится только как явно помеченный fallback;
   - найденные Tilda `#order`-ссылки использовать только если название продукта совпадает с текущей лекцией, иначе это ложная ссылка оплаты.
   - если пользователь подтверждает опубликованную Tilda-страницу как рабочий маршрут оплаты, фиксировать это в `telegram-followup-packet.json` с provenance и использовать конкретный URL/якорь карточки.
6. В пост добавить четыре постоянных Moonn-маршрута:
   - подростковый лагерь по психологии: `https://moonn.ru/podrostkovyy-lager-psihologiya`;
   - «Экзамены без паники» / психологическая подготовка к экзаменам: `https://moonn.ru/psypodgotovka1`;
   - онлайн-консультация: брать прямой `#order:Онлайн-консультация Татьяны Мунн =8000` с главной страницы, если он доступен;
   - оставить отзыв о Татьяне Мунн: `https://moonn.ru/otzivi?ostavit-otzyv=1&source=homepage_reviews_banner#moonn-review-funnel`.
7. Сохранить `telegram-followup-draft.md` и `telegram-followup-packet.json` в event package.
8. Отправить приватный preview в Telegram с фото и встроенными ссылками, сохранить proof-скриншот в event package и остановиться перед публикацией в канал.
9. После явного утверждения пользователя отправить пост в `https://t.me/moonn_official` только через подтверждённый Telegram-контур.
10. После отправки проверить пост в ленте канала: фото отображается, ссылки встроены в синий текст, нет raw URL в видимом тексте, нет Drive-ссылок.
11. Сохранить proof-скриншот опубликованного канального поста в event package, обновить `telegram-followup-packet.json`, `docs/codex-chat-history.md` и, при наличии, event manifest.

Правило веток:
- Для live-публикации сайта сначала определить deploy branch из `.github/workflows/pages.yml`.
- Если GitHub Pages публикует только `main`, то финальный scoped-коммит должен попасть в `main`, даже если подготовительная ветка была другой.
- Нельзя объявлять публикацию завершённой, пока не проверен фактический live URL.
