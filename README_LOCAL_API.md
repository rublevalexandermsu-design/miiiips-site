# Локальный API для прототипа МИИИИПС

Этот контур нужен, чтобы формы сайта работали не как заглушки, а как реальный UX-маршрут:
- запись заявки в Google Sheets;
- уведомление по email через Gmail API;
- сохранение локального JSONL-журнала;
- отдача новостей, курсов, видео и демо-данных для фронта.

## Как это работает
1. Запускается `local_api_server.py`.
2. Он одновременно:
   - раздаёт статические HTML-файлы сайта;
   - отвечает на `/api/health`, `/api/site-data`, `/api/forms`.
3. При отправке формы сервер:
   - пишет запись в `runtime/submissions.jsonl`;
   - добавляет строку в Google Sheets;
   - отправляет email или создаёт draft через Gmail API.

## Конфиг
Локальный конфиг хранится в `local_api_config.json` и не коммитится.
Шаблон лежит в `local_api_config.sample.json`.

## Запуск
Используйте уже знакомый батник `start_site_local_3007.bat`.
После обновления он должен поднимать именно этот сервер, а не обычный static-only http.server.
