# GitHub Pages Prep

Эта папка подготовлена как отдельный статический сайт для публикации через GitHub Pages.

Что уже сделано:
- основной файл входа: `index.html`;
- `.nojekyll` добавлен;
- есть workflow `.github/workflows/pages.yml`;
- локальные audit/runtime/config файлы исключены через `.gitignore`;
- формы и новости умеют работать через `prototype-enhancements.js` и локальный API-контур;
- при публикации на GitHub Pages фронт продолжит работать как статический сайт, а формы уйдут в fallback-режим, пока не появится постоянный внешний endpoint.

Локальный реалистичный режим:
- запускается `local_api_server.py`;
- формы пишутся в локальный JSONL;
- при наличии валидных Google-токенов возможны Google Sheets + Gmail.

Когда появится GitHub-аккаунт:
1. Создаем новый репозиторий под сайт.
2. Подключаем удаленный `origin`.
3. Пушим содержимое этой папки.
4. Включаем GitHub Pages из ветки `main`.

Ожидаемый результат:
- постоянная публичная ссылка;
- без временных туннелей;
- обновления сайта через обычный `git push`.
